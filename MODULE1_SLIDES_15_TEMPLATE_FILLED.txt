--------------------------------------------------------------------------------
SLIDE 2 — Pain → Bet → Payoff
--------------------------------------------------------------------------------
Problem
  Teams lack one auditable object for “what’s trending” on XHS—only screenshots, chats, and opinions.

Hypothesis
  IF we cluster and label real posts with config + LLM, THEN each run emits trend objects with post IDs and snippets anyone can verify.

Outcome
  Faster alignment on trends; measurable: posts in window, distinct trend objects per run, stable run logs (tune targets in Week 12).

Speaker notes: Memory line for the whole pitch; stay concrete.

SLIDE 3 — Core User Pain / Quote
--------------------------------------------------------------------------------

SLIDE 4 — Why Care?
--------------------------------------------------------------------------------
Time cost
  Hours per week reconciling screenshots, forwards, and ad-hoc search—not one shared artifact.

Revenue / performance impact
  Weaker follow-ups and missed moments when trend claims aren’t grounded in evidence.

Brand / operational risk
  In luxury, confident but ungrounded messaging erodes trust; star CA judgment doesn’t transfer without structured evidence.

Speaker notes: Directional impact only—no fake ROI.

SLIDE 5 — Constraints / Realism
--------------------------------------------------------------------------------
We cannot assume perfect APIs, unified shopper IDs, or stable XHS UI forever.

• No cross-silo shopper identity — public XHS + our scrape session
• Privacy: anonymized creators; post text as appropriate for trends
• Real batches in testing — proof corpus 197 posts (run_0022)

Today: batch scrape + JSON + Supabase trend/run logs
Future: official feeds or listening partners if LVMH negotiates access

Speaker notes: Execs will ask—show you assumed real limits.

--------------------------------------------------------------------------------
SLIDE 6 — Main Bet + Falsifier
--------------------------------------------------------------------------------
Main bet
  IF we cluster and label with rules + LLM on real posts, THEN each run produces trend objects with evidence—not invented narratives.

Falsifier
  If trends routinely lack verifiable post IDs or contradict source text, the approach fails (reviewers reject; evidence fields empty).

Interview insight (supports need — optional on slide):
  “应该是最后一个吧最后一个风格” — Reviewer C (Fendi), on style repetition vs noise

Speaker notes: Module-sized bet, not “solve clienteling.”

SLIDE 8 — Core Decision
--------------------------------------------------------------------------------
