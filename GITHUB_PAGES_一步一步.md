# 把网页公开到网上（给完全不会用 GitHub 的人）

你要的两个页面已经都在仓库里了，只差 **在 GitHub 网页上点几下**。下面按顺序做即可，**不用安装软件、不用写命令**。

---

## 你要打开的链接（做完下面步骤后才有用）

| 页面 | 链接 |
|------|------|
| 二十人卡片 | https://m-ny.github.io/mvp/ |
| 深色 Run Log | https://m-ny.github.io/mvp/runlog/ |

现在如果打开是 **404**，说明还没做完下面的「保存」——做完等 1～2 分钟再试。

---

## 第一步：打开仓库

1. 用浏览器打开（整段复制到地址栏）：  
   **https://github.com/m-ny/mvp**
2. 确认右上角是你**已登录**的 GitHub 账号（头像在右上角）。

---

## 第二步：进「设置」

1. 在仓库页面**最上面一排**标签里，找到并点击 **「Settings」**（设置）。  
   - 若看不到：把窗口拉宽一点；或点 **`⋯`** 更多里找 Settings。  
   - **注意**：必须是 **仓库的 Settings**，不是你自己账号头像里的 Settings。

---

## 第三步：打开 Pages

1. 在 Settings 页面**左侧很长的一列菜单**里，往下滚。  
2. 找到 **「Pages」**（在 **Code and automation** 那一组里），点一下。

---

## 第四步：选发布方式（最关键）

1. 中间有一块叫 **「Build and deployment」**。  
2. 在 **「Source」**（来源）下面，点开下拉框。  
3. 选 **「Deploy from a branch」**（从分支发布）——**不要**选「GitHub Actions」，除非你熟悉 Actions。

---

## 第五步：选分支和文件夹

1. **Branch** 选 **`main`**（一般已经默认是 main）。  
2. 右边文件夹选 **`/docs`**（不是 `/ (root)`）。  
3. 点 **「Save」**（保存）。

---

## 第六步：等一会儿再打开

1. 同一页往下滚，可能会看到 **「Your site is live at …」** 或 **「Visit site」**，点它也可以。  
2. 若暂时没有提示，**等 1～2 分钟**，再打开上面的两个链接。

---

## 若你「没有权限」点不了 Settings

说明这个仓库你不是管理员。需要 **让仓库管理员**（例如团队里负责 GitHub 的人）按上面 **第二～五步** 帮你点一次；或把你的账号加成 **Admin**。

---

## 完全不想碰 GitHub？（备选）

### A. 只给同事看自己电脑上的效果（同一 WiFi）

1. 让会一点终端的同事在项目里执行：  
   `cd docs && python3 -m http.server 8765`  
2. 同事告诉你一个形如 `http://192.168.x.x:8765/` 的地址，你手机连**同一个 WiFi** 打开。  
   - 看 Run Log：后面加 **`runlog/`**，例如 `http://192.168.x.x:8765/runlog/`

### B. 用网盘发「压缩包」给别人本地打开（不是公网链接）

把本机文件夹 **`docs/runlog`** 打成 zip 发给对方，对方解压后双击 **`index.html`**（有时浏览器会拦本地数据，不如 A 稳）。

---

## 还是不会？

把本文档链接发给能登录 **m-ny/mvp** 仓库的同事，说：「请按 **第四、五步** 选 **main + /docs** 保存一下。」即可。

文档在仓库里的路径：**`GITHUB_PAGES_一步一步.md`**（与本文件相同内容）。
